# 002 — 스크리너 확장 1차

001 위에 통계 심화(윌슨 하한·회복 소요일·MAE·보조지표), 일일 스냅샷과 신규 진입 뷰, 시장 레짐 배지, 종목 상세 페이지, 스파크라인, 탐색 인터랙션(검색·정렬·관심목록), 파라미터 프리셋, 수집 엔드포인트, 방법론·한계 페이지, 데이터 품질 패널을 추가한다.

## 핵심 정의 (측정 가능)

- **윌슨 하한**: 성공률의 윌슨 점수 구간 95% 하한(z=1.96) — 표본이 작을수록 성공률보다 크게 낮아진다
- **회복 소요일**: 성공 사례에서 진입일부터 목표(+5%) 최초 도달까지 거래일 수
- **MAE**: 진입일 이후 관찰 기간(20거래일) 내 최저 종가의 진입가 대비 하락률
- **신규 진입**: 직전 스냅샷에서 하락 상태가 아니었던 종목이 오늘 하락 상태가 된 경우
- **시장 레짐**: SPY의 하락 상태 여부(60거래일 고점 대비 −10%)
- **프리셋**: 보수(−15%/30일/+5%) · 기본(−10%/20일/+5%) · 공격(−7%/10일/+3%)

## 요구사항

R1 (Event): 사용자가 대시보드에 접속하면, 시스템은 하락 구간 행에 윌슨 하한·표본 수를 함께 표시해야 한다
R2 (Ubiquitous): 시스템은 항상 하락 구간을 윌슨 하한 내림차순으로 정렬해야 한다
R3 (Ubiquitous): 시스템은 항상 종목별 보조지표(MA50·MA200 이격, 볼린저 %B, 거래량 급증율)를 계산해 상세 페이지에 표시해야 한다
R4 (Event): 사용자가 대시보드에 접속하면, 시스템은 직전 스냅샷 대비 신규 하락 진입 종목을 별도 구간으로 보여야 한다
R5 (Ubiquitous): 시스템은 항상 기본 프리셋 분석 결과를 일일 스냅샷으로 SQLite에 저장해야 한다
R6 (Event): 수집 엔드포인트가 호출되면, 시스템은 유니버스 전체 시세를 강제 갱신하고 요약 JSON을 반환해야 한다 (CRON_SECRET 환경변수가 설정된 경우 Bearer 인증을 요구한다)
R7 (State): SPY가 하락 상태인 동안, 시스템은 시장 하락 국면 배지를 헤더에 표시해야 한다
R8 (Event): 사용자가 종목 상세 경로에 접속하면, 시스템은 2년 가격 차트에 과거 진입 시점 마커와 사례별 결과 표를 보여야 한다
R9 (Ubiquitous): 시스템은 항상 종목 행에 최근 60거래일 스파크라인을 표시해야 한다
R10 (Event): 사용자가 탐색 표에서 검색어·카테고리·정렬·관심목록 필터를 조작하면, 시스템은 페이지 이동 없이 행을 갱신해야 한다 (관심목록은 브라우저 로컬 저장)
R11 (Event): 사용자가 프리셋을 선택하면, 시스템은 해당 파라미터로 전 종목을 재계산해 보여야 한다
R12 (Ubiquitous): 시스템은 항상 방법론·한계 페이지(생존 편향, 레짐 한계, 비공식 데이터 출처, 변동성 잠식)를 제공해야 한다
R13 (Unwanted): 수집 오류가 누적된 경우에도, 시스템은 품질 패널에 종목별 오류 이력을 노출해야 한다
R14 (Invariant): 윌슨 하한은 표본 구성과 무관하게 0 이상이고 성공률 이하여야 한다

## 계약

수집 엔드포인트 (R6):

```
GET /api/collect[?force=1]
Authorization: Bearer <CRON_SECRET>   # CRON_SECRET 환경변수 설정 시에만 요구
200 { "ok": true, "collected": <int>, "failed": <int>, "durationMs": <int> }
401 { "ok": false, "error": "unauthorized" }   # 토큰 불일치
```

- force=1이면 24h TTL을 무시하고 전 종목 재수집한다. 기본은 TTL 준수.
- 응답의 collected+failed = 유니버스 크기. 실패 상세는 fetch_log(R13 품질 패널)로 조회.

## 시나리오

S1 (covers: R1,R2,R9,R14): Given 픽스처 대시보드 / When 접속하면 / Then 하락 구간 FIXDN 행에 성공률 100%와 윌슨 하한(34%)·스파크라인이 보이고 정렬은 윌슨 하한 내림차순이다
S6 (covers: R4,R5,R7): Given 직전 스냅샷에 FIXDN이 하락 아님으로 기록된 픽스처 / When 접속하면 / Then FIXDN이 신규 진입 구간에 나타나고 시장 레짐 배지(SPY 기준)가 표시된다
S7 (covers: R3,R8): Given 픽스처 / When /t/FIXDN에 접속하면 / Then 진입 마커가 있는 차트·사례 표 2행·보조지표 카드가 보인다
S8 (covers: R10): Given 탐색 표 / When 검색어로 FIXUP을 입력하면 / Then FIXUP 행만 남고, 관심 토글은 새로고침 후에도 유지된다
S9 (covers: R11): Given 대시보드 / When 공격 프리셋 링크를 선택하면 / Then 페이지가 −7%/10일/+3% 파라미터로 재계산된 표를 보여준다
S10 (covers: R12,R13): Given 픽스처 / When 방법론 페이지와 품질 패널을 열면 / Then 생존 편향 한계 서술과 FIXBAD 오류 이력이 보인다
S11 (covers: R6): Given 픽스처 / When GET /api/collect를 호출하면 / Then 200과 collected+failed=유니버스 크기 JSON이 반환된다

## [ASSUMED]

1. 제외(후속 볼트): 외부 알림(웹훅 — 시크릿 필요, 에스컬레이션 ③), 펀더멘털 병합(비공식 의존 확대), 점수·등급화(면책 수위 재설계 필요), 공개 JSON API(수집 엔드포인트 외).
2. 신규 진입 판정은 기본 프리셋 기준이며, 직전 스냅샷이 없으면(첫 가동) 신규 진입 구간은 비어 있다. 픽스처는 어제 스냅샷을 시드해 결정론을 확보한다.
3. 거래량 저장을 위해 SQLite 스키마 v2(prices에 volume 추가) — 기존 캐시 DB는 파생 데이터이므로 마이그레이션은 드롭 후 재생성(user_version=2).
4. 수집 크론은 Vercel cron(vercel.json, 평일 21:30 UTC) 가정 — 다른 배포면 외부 스케줄러가 R6 엔드포인트를 호출한다.
5. 탐색 인터랙션(R10)은 이 프로젝트 첫 클라이언트 컴포넌트 — 표 데이터는 서버 직렬화로 전달하고 관심목록은 localStorage 키 watchlist에 저장한다.
6. 정렬 기준 변경(R2: 성공률 → 윌슨 하한)은 001 R3의 정렬 정의를 대체한다 — 표본 부족 종목의 과대 노출을 막는다.
