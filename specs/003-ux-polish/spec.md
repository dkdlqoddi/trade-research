# 003 — UX 개선 일괄

002 위에 사용자 경험 개선 15항목을 적용한다: 로딩 스켈레톤, 모바일 카드 레이아웃, 용어 도움 링크, 탐색 상태 URL 유지, 방문 이후 변화 배지, 관심 구간 상단 고정, 하락 표 정렬, 상세 차트 강화(날짜 축·기간·거래량·마커 설명), 이웃 탐색, 비교 뷰, 빈 상태 맥락, 수동 갱신, 접근성(aria-sort·skip-link), CSV 내보내기, 테마·밀도 토글.

## 요구사항

R1 (Event): 분석 데이터 준비 전에 사용자가 접속하면, 시스템은 빈 화면 대신 스켈레톤 플레이스홀더를 보여야 한다
R2 (State): 뷰포트 너비가 720px 미만인 동안, 시스템은 주요 표를 카드형 행 레이아웃으로 보여야 한다
R3 (Ubiquitous): 시스템은 항상 통계 용어 헤더(윌슨 하한·MAE·%B·거래량 급증)에 방법론 페이지 해당 절로 가는 도움 링크를 제공해야 한다
R4 (Event): 사용자가 탐색 표의 검색어·카테고리·정렬을 바꾸면, 시스템은 그 상태를 URL 쿼리에 반영해 새로고침·공유 후에도 유지해야 한다
R5 (Event): 사용자가 재방문하면, 시스템은 마지막 방문 이후 하락 진입한 종목에 구분 배지를 보여야 한다 (방문 시각은 브라우저 로컬 저장)
R6 (State): 관심목록이 비어 있지 않은 동안, 시스템은 관심 종목 구간을 페이지 상단에 고정 표시해야 한다
R7 (Event): 사용자가 하락 구간 표의 열 머리글을 선택하면, 시스템은 그 열 기준으로 재정렬해야 한다
R8 (Event): 사용자가 상세 페이지에서 기간(6M·1Y·2Y)을 선택하면, 시스템은 날짜 축·거래량 서브차트·진입 마커 설명이 있는 차트를 그 기간으로 다시 그려야 한다
R9 (Ubiquitous): 시스템은 항상 상세 페이지에 하락 구간 정렬 순서 기준 이전·다음 종목 링크를 제공해야 한다 (하락 구간 밖 종목은 생략 가능)
R10 (Event): 사용자가 비교 경로에서 두 티커를 지정하면, 시스템은 두 종목의 통계·차트를 나란히 보여야 한다
R11 (Unwanted): 신규 진입이 없는 경우에도, 시스템은 비교 기준(직전 스냅샷 날짜)을 표시해 빈 구간의 맥락을 제공해야 한다
R12 (Event): 사용자가 갱신 버튼을 누르면, 시스템은 수집 엔드포인트를 호출하고 완료 후 화면을 다시 읽어야 한다
R13 (Ubiquitous): 시스템은 항상 본문 건너뛰기 링크를 제공하고, 정렬 가능한 열 머리글에 aria-sort 상태를 노출해야 한다
R14 (Event): 사용자가 내보내기 버튼을 누르면, 시스템은 현재 필터된 탐색 표를 CSV 파일로 내려받게 해야 한다
R15 (Event): 사용자가 테마(다크·라이트) 또는 표 밀도(기본·조밀)를 토글하면, 시스템은 즉시 반영하고 브라우저 로컬에 저장해야 한다

## 시나리오

S1 (covers: R1,R2,R13): Given 픽스처 대시보드 / When 접속하면(720px 미만 뷰포트 포함) / Then 건너뛰기 링크가 존재하고 본문이 로드되며 표가 카드형 행으로 바뀐다 (스켈레톤은 라우트 loading 파일 존재로 보장)
S2 (covers: R7,R3): Given 하락 구간 표 / When 성공률 머리글을 선택하면 / Then aria-sort가 바뀌며 재정렬되고, 용어 도움 링크가 방법론 앵커를 가리킨다
S3 (covers: R4): Given 탐색 표 / When 검색어를 입력하면 / Then URL 쿼리에 반영되고 새로고침 후에도 검색 상태가 유지된다
S4 (covers: R5,R6): Given 관심 등록과 과거 방문 시각이 저장된 상태 / When 페이지를 다시 열면 / Then 상단 관심 구간이 보이고 방문 이후 진입 배지가 나타난다
S5 (covers: R8): Given /t/FIXDN?range=1y / When 페이지를 열면 / Then 날짜 축 라벨·거래량 서브차트·마커 설명(title)이 있는 차트가 보인다
S6 (covers: R9,R10): Given 하락 구간에 둘 이상 종목 / When 상세와 /compare를 열면 / Then 이전·다음 링크와 두 종목 나란히 보기가 동작한다
S7 (covers: R11,R12,R14,R15): Given 픽스처 대시보드 / When 신규 진입 구간·갱신 버튼·CSV 버튼·테마 토글을 확인하면 / Then 직전 스냅샷 날짜가 표기되고, CSV가 내려받아지며, html 테마 속성이 바뀐다

## [ASSUMED]

1. 방문 시각·관심·테마·밀도는 전부 브라우저 localStorage — 서버 계정 없음(개인화 서버 저장은 후속).
2. 탐색 상태 URL 반영은 history.replaceState — Next 네비게이션을 타지 않아 서버 재계산이 없다.
3. 테마 초기 적용은 클라이언트 마운트 시 — 첫 페인트 짧은 다크 플래시 허용(인라인 스크립트 회피).
4. 차트 기간: 6M=최근 126·1Y=252 거래일, 2Y=전체. 거래량 서브차트는 동일 SVG 하단 18%.
5. 이웃 탐색은 기본 프리셋 하락 정렬 기준 — 하락 구간 밖 종목 상세에는 대시보드 복귀 링크만.
6. CSV는 UTF-8 BOM 포함(엑셀 한글 호환), 현재 필터·정렬이 반영된 행만.
